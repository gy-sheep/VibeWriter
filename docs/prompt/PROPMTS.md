## IDEAS
- [] 생성된 글들이 너무 어색해서 quality.py의 수정이 필요함, ChatGPT, 제미나이와 크로스 체크를 통해 보완해볼것
LLM은 런타임에 전달되는 프롬프트 문자열만 봅니다. 
DESIGN_SPEC.md는 사람이 읽는 설계   문서일 뿐, LLM에게 전달되지 않습니다.                                                  
                                                                                         
  DESIGN_SPEC.md        →  개발자가 읽고 코드/프롬프트에 반영                            
                                      ↓                                                  
  writer.py _SYSTEM_PROMPT     →  WriterAgent LLM이 참고
  quality.py _POLISH_SYSTEM    →  QualityAgent LLM이 참고
  humanize.py 코드 규칙        →  코드가 직접 처리

  즉, DESIGN_SPEC.md의 정책이 LLM에 전달되려면 개발자가 프롬프트에 직접 옮겨써야 합니다.

  현재 상태에서 DESIGN_SPEC의 내용 중 프롬프트에 반영된 것과 빠진 것을 보면:

  반영된 것 (writer.py or quality.py 프롬프트에 있음)
  - AI 과잉 표현 금지
  - 문장 길이 단조로움 금지
  - 단계식 구조화 금지

  빠진 것 (DESIGN_SPEC에는 있지만 프롬프트에 없음)
  - 미세한 비논리 허용
  - 애매함과 확신을 섞기
  - 불완전한 구조 유지
  - 개인적 관점의 편향 허용
  - 추상어 남용 금지 ("효율성", "중요성" 등)
