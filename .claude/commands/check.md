신규 구현 또는 수정된 코드에 대해 방어 코드 체크리스트를 실행한다. 아래 순서를 따른다.

## 분석 단계

1. `git diff HEAD` 를 실행해 변경된 `.py` 파일과 실제 변경 내용을 파악한다.
   - `.py` 파일 변경이 없으면 "점검할 코드 변경사항이 없습니다"를 출력하고 종료한다.

2. 변경된 각 `.py` 파일을 읽고 아래 6개 항목을 기준으로 점검한다.

   | 항목 | 확인 내용 |
   |------|----------|
   | 입력값 검증 | 빈 문자열·None·공백 전용 입력에 대한 early return 처리 |
   | 파일 I/O | `open`, `read_text`, `write_text`, `mkdir` 등 모든 파일 작업을 `try/except OSError`로 감쌈 |
   | LLM 응답 파싱 | JSON 파싱 실패·빈 응답·예상 외 형식에 대한 fallback 처리 |
   | 예외 전파 | 내부 함수의 예외가 상위 호출자까지 비처리 상태로 전파되지 않는지 확인 |
   | 정규식·문자열 매칭 | 부분 문자열 오탐 방지 — 카테고리·키워드 매칭은 완전 단어(`\b`) 기준 |
   | 외부 호출 실패 | httpx/Ollama/Gemini 등 외부 의존 호출은 항상 `except` + 로그 + fallback 처리 |

## 결과 보고

3. 항목별로 아래 형식으로 결과를 출력한다.

   ```
   파일: agents/xxx.py
   ✅ 입력값 검증 — early return 처리 확인
   ✅ 파일 I/O — OSError 처리 확인
   ⚠️  LLM 응답 파싱 — line 42: json.loads 실패 시 fallback 없음
   ...
   ```

4. 미비한 항목이 있으면 수정 여부를 사용자에게 확인한다. 승인 시 해당 파일을 수정한다.
