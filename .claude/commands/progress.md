커밋 없이 PROGRESS.md만 업데이트한다. 세션 종료 시 진행 상태를 저장할 때 사용한다. 아래 순서를 따른다.

## 분석 단계

1. `PROGRESS.md` 와 `HISTORY.md` 를 읽어 현재 상태를 파악한다.

2. `git diff HEAD` 와 `git status` 를 실행해 이번 세션에서 변경·완료된 내용을 파악한다.

3. 아래 기준으로 업데이트할 내용을 정리한다.
   - 이번 세션에서 **완료된 작업**: PROGRESS.md의 완료 상태 섹션에 추가 → HISTORY.md로 이동 준비
   - **다음 작업**: 새로 확인된 다음 단계로 업데이트
   - **프로젝트 구조**: 신규 파일·모듈 추가 시 반영

## 제안 단계

4. 변경할 내용을 아래 형식으로 사용자에게 보여주고 승인을 받는다.

   ```
   [PROGRESS.md 변경 내용]
   - 완료로 이동: ...
   - 다음 작업 업데이트: ...
   - 기타: ...

   [HISTORY.md 이동 내용] (없으면 "없음")
   - ...
   ```

## 실행 단계 (승인 후)

5. `PROGRESS.md` 를 업데이트한다.
   - 완료된 항목은 해당 섹션으로 이동하고 ✅ 표시
   - 다음 작업 섹션을 최신 상태로 정리

6. 완료된 Phase/Step이 있으면 관련 내용을 `HISTORY.md` 로 이동한다.

7. 업데이트 완료를 보고한다.
