커밋을 진행한다. 아래 순서를 따른다.

1. `docs/git/COMMIT_CONVENTION.md`를 읽어 커밋 메시지 규칙을 확인한다.
2. `git status`와 `git diff`를 실행해 변경 사항을 파악한다.
3. `git diff`에 `.py` 파일 변경이 포함된 경우에만 `/sync-docs` 를 실행해 관련 문서를 업데이트한다. 문서·설정 파일만 변경된 경우 이 단계를 스킵한다.
4. `PROGRESS.md`를 읽고 변경 사항이 현재 상태에 반영되어야 하는지 판단한다.
   - 반영이 필요하면 업데이트 내용을 사용자에게 제안하고 승인을 받은 후 업데이트한다.
   - 완료된 작업은 `HISTORY.md`로 이동하고 `PROGRESS.md`는 다음 작업 기준으로 정리한다.
5. 규칙에 맞는 커밋 메시지를 작성하고 사용자에게 보여준 뒤 승인을 받는다.
6. 승인되면 커밋을 실행한다.
7. 커밋 완료 후 push 여부를 사용자에게 확인한다.
8. push 승인 시 현재 브랜치 기준으로 `git push`를 실행한다.
