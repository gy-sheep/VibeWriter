변경된 코드와 관련된 문서를 정밀하게 분석하고 업데이트한다. 아래 순서를 따른다.

1. `git diff HEAD` 와 `git status`를 실행해 변경된 파일 목록과 실제 diff 내용을 파악한다.

2. 변경된 각 소스 파일에 대해 다음을 분석한다.
   - 추가·수정·삭제된 함수, 클래스, 설정값, 파일 경로
   - 동작 방식이 바뀐 로직 (예: 선택자 추가, 재시도 로직 변경 등)

3. `docs/` 디렉터리 전체를 탐색해 변경된 내용을 참조하는 문서를 찾는다.
   - 파일명, 함수명, 설정 키, 경로, 기능 설명 등을 기준으로 검색
   - 대상 문서: `docs/dev/`, `docs/roadmap/DESIGN_SPEC.md` 등

4. 각 문서를 읽고 실제 코드와 불일치하는 항목을 구체적으로 식별한다.
   - 존재하지 않는 함수·설정 설명
   - 누락된 기능 설명
   - 잘못된 파일명·경로·파라미터

5. 불일치 항목을 사용자에게 목록으로 보여주고 업데이트 승인을 받는다.

6. 승인되면 해당 문서들을 업데이트한다.
   - 코드에 없는 내용은 제거하거나 수정
   - 코드에 새로 추가된 내용은 문서에 반영
   - 문서의 기존 구조와 문체를 유지

7. 업데이트된 문서 목록과 변경 내용을 사용자에게 보고한다.
